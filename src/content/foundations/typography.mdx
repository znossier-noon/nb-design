## Typeface

The Branding Figma source defines the primary type family as **noontree**.
Use it for Noon Business brand, product and documentation typography.

The web implementation prefers `noontree` when available and falls back to the
current loaded sans font until the production noontree font files are available
in the repo.

Use **JetBrains Mono** only for code, tokens, order IDs, financial figures and
tabular data.

## Headings

Headings use negative tracking from the Figma tokens. Use one dominant page title,
then step down through H1, H2, H3 and H4 without skipping hierarchy.

## Labels

Labels are compact, high-utility styles for product surfaces. Use them for
buttons, tabs, chips, metadata labels and status markers. Keep labels short and
action-oriented.

## Body

Body styles use zero tracking and tighter line heights than long-form editorial
prose. Use Body1 for primary product copy, Body2 for supporting text and table
content, and Body3 for captions or legal support.

## Tiny

Tiny is reserved for microcopy, counters and dense metadata. Tiny Extended uses
`1px` tracking and uppercase for overlines and section markers.

## Arabic Typography

Arabic typography should match the same hierarchy and spacing logic as English:
clear titles, practical body copy and generous line height for readability. Use
Arabic-first layouts for Arabic pages instead of forcing translated text into
English compositions.

## Loading

The Figma source family is `noontree`. Production should self-host approved
noontree files when they are available. Do not load brand fonts from third-party
CDNs in product code.

```css
:root {
  --font-sans: "noontree", var(--font-figtree), ui-sans-serif, system-ui, sans-serif;
}
```

```tsx
<h1 className="text-[44px] leading-[48px] tracking-[-0.8px] font-extrabold">
  Page title
</h1>
<p className="text-[16px] leading-[20px]">Body1 copy...</p>
<span className="font-mono text-[13px]">NB-20581</span>
```

Latin product copy uses **NoonTree**. Arabic uses **Noon Arabic**. Hierarchy
comes from size and weight, never from mixing families on the same line.

## Type scale

Two registers per script: **editorial** (Display XL through Headline) is fluid
via `clamp()`; **interface** (Title and below) uses fixed sizes. Toggle EN
and AR to compare metrics.

<TypeScale />

## Typefaces

| Family | Script | Role | Weights | Glyphs |
| --- | --- | --- | --- | --- |
| NoonTree | Latin | Display, headings, UI, body | 400, 500, 600 | `dhm`, `sar` |
| Noon Arabic | Arabic | Display, headings, UI, body | 400, 500, 600 | `dhm`, `sar` |
| Noon Mono | Latin | Code, tokens, tabular IDs | 400, 500 | `dhm`, `sar` |

**NoonTree** is Figtree repackaged for Noon: same metrics as the upstream family.
**Noon Arabic** is Cairo under the brand name: the RTL companion with the same
weight scale (400, 500, 600). **Noon Mono** is JetBrains Mono under the brand
name: tabular figures for code and data. All three families ship the brand
currency glyphs.

All three load via `next/font` with `display: swap` (self-hosted by Next.js). Pin
local woff2 files in `public/fonts/noontree/`, `public/fonts/noon-arabic/`, and
`public/fonts/noon-mono/` when you need frozen builds. Never load fonts from
third-party CDNs in product code.

## Arabic rules

- Set `lang="ar"` and `dir="rtl"` on the page or section root
- Use `font-arabic` or rely on `:lang(ar)` — never NoonTree for Arabic copy
- **No letter-spacing** on Arabic. Tracking sliders do not apply
- Line heights run **looser** than Latin (see Arabic column in the type scale)
- Overlines stay small semibold text; avoid forced ALL CAPS in Arabic
- Mixed Latin + Arabic in one line: wrap each run in its own `lang` span

  </Do>
  <Dont>
    Apply Latin negative tracking to Arabic, or use NoonTree for Arabic sentences.
  </Dont>
</DoDont>

## Usage rules

### Hierarchy

A screen should read as a clear outline when you squint. If two adjacent text
blocks compete, demote one.

<DoDont>
  <Do>
    Use one Headline per page, and step down through Title and Body without
    skipping more than one level.
  </Do>
  <Dont>
    Create hierarchy with color or italics, weight and size are the only
    hierarchy tools in this system.
  </Dont>
</DoDont>

### Badges and overlines

Two small-type registers. Do not mix them.

| Register | Case | Tracking | Font | Use for |
| --- | --- | --- | --- | --- |
| **Overline** | ALL CAPS (Latin only) | `0.08em` (`text-overline`) | NoonTree semibold | Sidebar groups, TOC, section labels |
| **Badge** | Sentence case | Normal | NoonTree / Noon Arabic semibold | Product tags: Paid, New, Active |
| **Status** | Sentence case | Normal | Noon Mono semibold | Doc platform: WIP, Done, Planned |

Never set badges or status labels in all caps. Color and the pill shape carry the emphasis.

### Line length and rhythm

- Latin large text tracks tight (Display XL −3.5%), small text needs room (Caption +1%)
- Arabic: zero tracking everywhere; body line height 1.85, dense UI 1.6
- Body text: 45–75 characters per line (Latin), ~50–70 characters (Arabic)
- Never justify text

### Numbers and data

Any column of numbers, order IDs, amounts, timestamps, is Noon Mono
with tabular figures. Currency uses the brand glyphs (`dhm`, `sar`) via their
Unicode codepoints in whichever family matches the surface:

```tsx
<span>{'\u20C3'} 1,234.50</span> {/* AED, glyph dhm */}
<span>{'\u20C1'} 1,234.50</span> {/* SAR, glyph sar */}
```

<DoDont>
  <Do>
    Right-align numeric table columns and set them in mono with tabular
    figures.
  </Do>
  <Dont>
    Mix mono identifiers into running prose, only data surfaces get mono.
  </Dont>
</DoDont>

## Engineering reference

Latin and Arabic share utility class names. Arabic metrics apply under
`lang="ar"`, `dir="rtl"`, or the `type-arabic` class.

```css
body {
  font-family: var(--font-noontree), ui-sans-serif, system-ui, sans-serif;
}

:lang(ar) {
  font-family: var(--font-noon-arabic), sans-serif;
  letter-spacing: 0;
}
```

```tsx
<html lang="ar" dir="rtl" className="type-arabic">
  <h1 className="text-headline text-ink">نظام التصميم</h1>
</html>

<h1 className="text-headline text-ink">Page title</h1>
<p className="text-body-lg text-ink-secondary">Lede paragraph…</p>
<span className="font-mono text-[13px]">NB-20581</span>
```
